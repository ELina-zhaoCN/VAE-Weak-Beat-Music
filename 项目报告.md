## 摘要

本项目实现了一个创新的变分自编码器(VAE)音乐生成系统，该系统能够从弱节拍音乐（如Ambient、Drone、Experimental等风格）中学习，并通过自定义的节拍正则化损失函数，自动生成具有规律节拍的音乐。核心创新点在于引入了基于自相关的节拍损失函数，使模型能够"看到高损失并自我调整"，最终输出节拍规律清晰的音乐作品。

## 2. 模型设计

### 2.1 整体架构

**模型类型**: 卷积变分自编码器 (Convolutional VAE)

**文件位置**: `vae_model.py`

**输入输出**:
- 输入: Mel频谱图 (batch, 1, 128, 431)
- 潜在空间维度: 128
- 输出: 重构Mel频谱图 (batch, 1, 128, 431)

### 2.2 编码器 (Encoder)

**目标**: 将输入压缩到128维潜在空间

**架构**:
```python
输入: (1, 128, 431)
  ↓
Conv2d(1→32, kernel=4, stride=2, padding=1)
  + BatchNorm2d(32)
  + ReLU
  → (32, 64, 215)
  ↓
Conv2d(32→64, kernel=4, stride=2, padding=1)
  + BatchNorm2d(64)
  + ReLU
  → (64, 32, 107)
  ↓
Conv2d(64→128, kernel=4, stride=2, padding=1)
  + BatchNorm2d(128)
  + ReLU
  → (128, 16, 53)
  ↓
Flatten
  → (108,544)
  ↓
Linear(108544 → 128)  # mu (均值)
Linear(108544 → 128)  # logvar (对数方差)
```

**特征**:
- 3层卷积网络
- 每层后接BatchNorm和ReLU
- 逐步降采样（步长=2）
- 最终展平维度: 128 × 16 × 53 = 108,544

**重参数化技巧**:
```python
z = μ + σ × ε
其中: σ = exp(0.5 × log(σ²))
     ε ~ N(0, 1)
```

### 2.3 解码器 (Decoder)

**目标**: 从128维潜在向量重构频谱图

**架构**:
```python
输入: z (128)
  ↓
Linear(128 → 108,544)
  ↓
Reshape → (128, 16, 53)
  ↓
ConvTranspose2d(128→64, kernel=4, stride=2, padding=1)
  + BatchNorm2d(64)
  + ReLU
  → (64, 32, 106)
  ↓
ConvTranspose2d(64→32, kernel=4, stride=2, padding=1)
  + BatchNorm2d(32)
  + ReLU
  → (32, 64, 212)
  ↓
ConvTranspose2d(32→1, kernel=4, stride=2, padding=1)
  → (1, 128, 424)
  ↓
Conv2d(1→1, kernel=3, stride=1, padding=1)
  → (1, 128, 424)
  ↓
Padding (424→431)
  → (1, 128, 431)
```

**特征**:
- 3层转置卷积网络
- 逐步上采样恢复分辨率
- 最后填充以匹配原始尺寸

### 2.4 模型参数

**总参数量**: 约50-60百万
- 编码器参数: ~27百万
- 解码器参数: ~27百万
- 全部可训练

**内存占用**:
- 模型大小: ~200-300 MB
- 批次大小=16时GPU内存: ~4 GB

## 3. 损失函数设计

### 3.1 总损失函数

**公式**:
```
Total Loss = Reconstruction Loss + β × KL Divergence + γ × Beat Loss
```

**文件位置**: `train.py` 和 `beat_loss.py`

### 3.2 重构损失 (Reconstruction Loss)

**目的**: 确保模型能够准确重构输入

**公式**:
```python
L_recon = MSE(reconstruction, original)
       = (1/N) Σ (x_recon - x_orig)²
```

**特点**:
- 使用均方误差(MSE)
- 逐像素比较频谱图
- 批次平均

### 3.3 KL散度损失 (KL Divergence)

**目的**: 正则化潜在空间，使其遵循标准正态分布

**公式**:
```python
L_KL = -0.5 × Σ(1 + log(σ²) - μ² - σ²)
```

**作用**:
- 防止潜在空间崩溃
- 确保生成能力
- 平滑潜在空间结构

### 3.4 节拍损失 (Beat Loss) ⭐ **核心创新**

**目的**: 强制模型生成具有规律节拍的音乐

**文件**: `beat_loss.py`

**实现流程**:
```python
def beat_loss(mel_spectrogram):
    # 1. 提取低频部分（鼓/贝斯频率，0-250 Hz）
    low_freq = mel_spectrogram[:, :, :10, :]  # 前10个Mel频带
    
    # 2. 计算能量包络
    energy = torch.mean(low_freq, dim=2)  # (batch, n_frames)
    
    # 3. 归一化到[0,1]
    energy = (energy - min) / (max - min)
    
    # 4. 基于FFT计算自相关
    autocorr = IFFT(|FFT(energy)|²)
    
    # 5. 找最大相关值（排除lag=0和前5个lag）
    max_corr = max(autocorr[:, 5:])
    
    # 6. 损失 = 最大相关值
    return max_corr.mean()
```

**工作原理**:
- **规律节拍**: 高自相关 → 高max_corr → **高损失**
- **弱节拍**: 低自相关 → 低max_corr → **低损失**
- **模型学习**: 为了降低总损失，需要**增加**节拍规律性！

**自相关计算**（Wiener-Khinchin定理）:
```python
R(τ) = IFFT(|FFT(x)|²)
```
- 复杂度: O(n log n)
- 完全可微分
- GPU加速

**关键参数**:
- `n_low_freq_bands`: 10（低频带数量）
- `exclude_lags`: 5（排除的初始lag数）
- `normalize`: True（归一化能量包络）

### 3.5 渐进式预热策略

**目的**: 防止KL崩溃和训练不稳定

**预热调度**:
```python
# 前20个epoch线性增加
β(epoch) = β_target × (epoch / 20)  # KL权重: 0 → 1.0
γ(epoch) = γ_target × (epoch / 20)  # 节拍权重: 0 → 0.5

# 20 epoch后: 固定权重
β = 1.0
γ = 0.5
```

**时间表**:
| Epoch | β (KL) | γ (Beat) | 训练阶段 |
|-------|--------|----------|---------|
| 0-5   | 0.00-0.25 | 0.00-0.125 | 学习基本重构 |
| 5-10  | 0.25-0.50 | 0.125-0.25 | 引入正则化 |
| 10-20 | 0.50-1.00 | 0.25-0.50 | 渐进节拍学习 |
| 20+   | 1.00 | 0.50 | 完整训练 |

### 4.2 训练流程

**单个epoch流程**:
```python
1. 对于每个批次:
   a. 加载数据到GPU
   b. 前向传播: reconstruction, mu, logvar = model(batch)
   c. 计算损失:
      - recon_loss = MSE(reconstruction, batch)
      - kl_loss = KL_divergence(mu, logvar)
      - beat_loss = beat_loss(reconstruction)
      - total = recon + β×kl + γ×beat
   d. 反向传播: total_loss.backward()
   e. 梯度裁剪: clip_grad_norm_(params, 1.0)
   f. 优化器步进: optimizer.step()
   g. 记录指标

2. 验证集评估:
   - 计算所有损失指标
   - 可选：BPM准确度评估
   
3. 检查点保存:
   - 保存最佳模型（最低验证节拍损失）
   - 每N个epoch保存检查点
   - 保存训练历史
```

### 4.5 "看到高损失并调整"机制

**工作原理**:
```
输入: 弱节拍音乐（Ambient）
  ↓
初始重构: 类似输入（弱节拍）
  ↓
节拍损失: 低（~0.1-0.2）
  ↓
但随着γ增加，低节拍损失在总损失中权重增大
  ↓
梯度反向传播
  ↓
解码器权重调整，生成更规律的节拍模式
  ↓
节拍损失增加（~0.6-0.7）
  ↓
更高的自相关 = 更规律的节拍 ✓
```

**梯度流**:
```
Beat Loss (对弱节拍为低)
    ↓ ∂L/∂θ
解码器权重
    ↓ 调整
输出更规律的时间模式
    ↓
自相关增加
    ↓
节拍损失增加 ✓
```
